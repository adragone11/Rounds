# Session: 2026-04-22 (Pip Web)

> **Scope:** All changes below are in `pip-web` (Vite + React + TypeScript). None of this ships to the mobile app (Pip, React Native + Expo). Mobile consumes the same Supabase `clients` table but doesn't share placements, recurrence meta, or Transition state.

## What We Built

### Message tone templates (Transition copy)
Four preset templates for the client text message, selectable via chips in the Transition header. Lives as a global pref keyed by `pip-transition-template` in localStorage.

- **Warm** — fastest/friendliest
- **Pricing** — "keep my rates steady" business reason
- **Reliability** — "on-time arrivals" professional
- **Transparent** — longest, fully honest

Each template takes `(firstName, dayFullName)` and produces the message. Every card's Copy button uses the active template at render time — so switching the chip updates every client's message instantly.

Framing rule: "Your cleaning is going to be on Thursdays" (about the appointment), not "I'm moving you to Thursdays" (about the person). Avoids the "asking permission" tone.

### Transition persistence (keyed by applyId)
Each Apply generates a UUID (`applyId`) stamped on both the Undo snapshot AND the Transition state. Keyed persistence means a fresh Apply automatically orphans stale Transition progress — no cross-session bleed.

- `Schedule.tsx` reducer state saves to `pip-transition-context`
- `TransitionView` internal state saves to `pip-transition-state-<applyId>`
- `CLOSE_TRANSITION` just hides — doesn't destroy data
- Transition button toggles open/close
- "Start fresh" / "Finish" / "Undo apply" are the only explicit wipes

Sidebar now survives reloads, tab close, multi-day workflows. User can close their laptop on Monday with 7/44 resolved, open Friday, pick up at 7/44.

### Undo Apply kept (not removed)
Originally planned to delete it. Kept it instead — it's the safety net for "oops I hit Apply while exploring Schedule Builder." Now includes a warning count: "Undo will discard 7 Transition confirmations."

Apply warning modal copy updated to advertise the undo option up front: "You can undo this anytime until you finish Transition."

### Undo swap (symmetric unwind)
When a swap commits, both cards get a `preSwapSnapshot` capturing their original `suggestedDay` / `targetRotation` / `reason` / `suggestedMessage`. "Undo swap" link appears on either card in the pair.

Undoing:
- Restores both to pre-swap state
- Flips both to `to-ask`
- Clears swap tags + snapshot
- If either was Confirmed post-swap, un-places them (simplest path; user's texts cover whether the client already agreed)

This pairs with Revert to To Ask (single-card undo) — both coexist.

### Smart Placement gate (≥1 placed neighbor)
With zero placed clients, every day scored the same and the ranking was noise. Smart Placement now shows a hint until ≥1 other client is on the calendar.

### Working hours + time budget (Schedule Builder)
New config row: start/end time inputs (default 8:00–17:00 = 9hr window). Sum of job durations per day must fit the window. Drive time intentionally not counted — location grouping makes it negligible.

- `OptimizeConfig` gained `workingMinutes`
- `peakMinutes` check runs alongside `peakCount` in `buildScheduleFromClusters`
- When a day overflows, VROOM tries the next-best day; if nothing fits, client goes to the Bench with a time-based reason
- Smart Placement already had this via `dateFits` — now uses the builder window instead of hardcoded 8h

No per-job time slots, no drive padding, no buffers. Keep it simple.

### UI fix — home drive chip clipping
The 🏠 14m chip above the first card was getting clipped by the scroll container. Moved inside the top of block0 as a floating white badge. Visible now.

## Issues Found & Fixed
- Exit modal lied about saving progress (said "you can resume later" but blew away state on close). Now accurate.
- Transition button was one-way (only opened). Now toggles.
- `CLOSE_TRANSITION` was destroying `transitionMoves: []`. Preserved now.
- "Close enough — finish" banner fired with 0m savings while 42 clients unresolved — removed (earlier session, now reconfirmed correct).

## Decisions Made
- **Per-day windows deferred** — single window for v1; extend later if users ask
- **Drive time NOT counted in budget** — grouping by location assumes travel is negligible within a day
- **Undo swap restores to To Ask, not Confirmed** — user's own texts track "did they already say yes"; reverting to Confirmed would require storing more state
- **Kept Undo Apply** — safety net for accidental Apply
- **No Ctrl+Z undo stack** — Revert + Undo swap covers the important cases
