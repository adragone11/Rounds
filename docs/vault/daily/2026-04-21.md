# Session: 2026-04-21

## What We Built

### Symmetric Swap in Transition (V2)
Rebuilt the "Can't Move" flow as a two-sided trade. When a client rejects their assigned day, the user picks a partner on another day and **both clients swap days**. The rejected client takes the partner's day; the partner takes the rejected's day. Biweekly A/B rotations flip as part of the trade where needed (`rotationShifts` flag).

- Swap candidate picker shows one best client per other day (closest to rejected by drive time, same cadence)
- Shows full day name + cadence label (`Weekly`, `Wk 1,3`, `Wk 2,4`) + drive time
- "rot shift" badge when biweekly rotation changes
- "re-ask needed" badge when the partner was already confirmed (they get un-placed back to To Ask)
- "Trade" button commits both moves; both cards flip to `to-ask`, unlock, and gain an `updated` badge (iteration bump)

### Revert to To Ask
Any resolved card (confirmed or cant-move) has a **Revert to To Ask** link. Calling it:
- If confirmed: un-places the client (clears recurrence, drops future placements, keeps frozen past)
- Resets card to `to-ask` status, `locked=false`, sends it back to the Pending section
- Fixes both accidental taps AND the swap-partner case (partner gets reverted automatically on trade)

### Commit Model Transform
Transition is no longer a notify-list ÔÇö it's the actual commit mechanism. Apply = freeze past, clear all recurrences. Each Transition confirm re-anchors one client individually via `reanchorClient`. Past is frozen, future is empty until confirmed.

### Carryover Section
Unchanged clients (Sarah Thu ÔåÆ Thu) used to be orphaned because `result.changes` excluded same-day assignments. Fix: build moves from ALL assignments; unchanged ones go into a "Carryover" section with a bulk "Confirm all" action.

### UI Polish
- Cadence labels on swap candidates (`Weekly`, `Wk 1,3`, `Wk 2,4`)
- "Swap withÔÇª" ÔåÆ **"Trade day withÔÇª"**

## Issues Found & Fixed
- **Swap picker empty** ÔÇö rejected was in `openingDayOccupants`, double-counted by capacity check. Fix: dropped capacity check; biweekly peak is `ceil(n/2)` not raw count anyway
- **Orphaned unchanged clients** ÔÇö `result.changes` excluded same-day. Fix: feed all assignments, separate into Carryover
- **Stale Transition state across applies** ÔÇö removed localStorage persistence entirely; fresh apply = fresh state
- **Partner reuse in swaps** ÔÇö added `swapPartnerClientId` tag; same client can't be a swap partner twice
- **Biweekly default in Schedule Builder** ÔÇö clients without recurrence now default to biweekly, not weekly
- **Vercel build fail** ÔÇö unused imports (`triggerReoptimize`, `SwapCandidate`, `useCallback`, `setLoading`)

## Decisions Made
- **Best-per-day** swap scoring over top-N or hybrid ÔÇö guarantees day diversity, prevents one dense day hogging slots
- **Symmetric trade** over asymmetric fill ÔÇö matches the real-world ask ("would Thursday work instead?")
- **No persistence on Transition state** ÔÇö every Apply is a fresh proposal
- **Revert on every resolved card** ÔÇö low-cost safety net for accidental confirms
