# Session: 2026-04-08

## What We Built
- **Google Places Autocomplete** — replaced all plain text address inputs with Google Places suggestions. Addresses now come with guaranteed lat/lng from Google.
- **Geocode status pins** — green pin = geocoded, yellow = not geocoded, grey = no address. Shows on sidebar and Clients page.
- **Driving route visualization** — click a day on calendar, see actual road-following route on Apple MapKit map via ORS Directions API.
- **Apply to Schedule button** — confirmed optimization moves can now actually be applied to the schedule.
- **Map view filtering** — day view shows only that day's clients + auto-draws route. Week view shows that week's clients.
- **Map deselect fix** — panning/zooming no longer clears selection. Added X button for manual deselect.
- **Recurrence-aware drag-drop** — moving clients between days preserves their weekly/biweekly/monthly pattern.
- **Frequency badges** — W, 2W, M, 1x badges on client cards in sidebar.
- **Optimizer fixes** — max jobs enforced in swaps, top 3 swaps per client cap, VROOM fingerprint caching.
- **Shuffle/Clear tools** — for testing schedule layouts with configurable days and max jobs.

## Issues Found & Fixed
- [[ORS geocoding unreliable]] — ORS silently fails, leaving clients at null coords
- [[VROOM non-deterministic]] — different suggestions each run, fixed with roster fingerprint caching
- [[Swap capacity bug]] — swaps didn't check max jobs per day
- [[Recurrence lost on drag-drop]] — moving between days created one-time instead of preserving frequency
- [[Unplaced detection month-bound]] — biweekly/monthly clients showed as unplaced in off-months

## Decisions Made
- Google Places API key is client-side (VITE_ prefix) — secured via referrer restrictions
- ORS kept as fallback for bulk import geocoding
- Route visualization uses ORS Directions API, not straight lines
- VROOM hypothesis cached per roster fingerprint, invalidated on client add/remove/address change
- Swaps capped at 3 per client to keep suggestions actionable
